Tutorial

 

Following is a tutorial of the major parts needed for executing BFOConvert:

 

IRI mapping file:

An IRI mapping file is a text file that is used to update IRI. The requirements of the file include:

  • The first line should be a header that lists the header information for the columns.
  • At least two columns are required: the first column represents the old IRIs, and the second column represents the new IRIs.
  • More columns can be added. For example, the BFOConvert includes a third column that represents the labels of the IRIs.
  • The default header cleanup file is HERE.

 

Header Cleanup File:

The file is an optional text file and used for final OWL output cleanup. Specifically, when a header cleanup file is provided, BFOConvert will remove/add/replace xmlns and/or OWL imports heads to the output OWL file based on the information provided in the header file.

The requirements of a header cleanup file include:

  • The first line should be a header that lists the header information for the columns.
  • The first column is the action to be performed on the output. Currently supported options include: add, remove, replace.
  • The second column is the type of a header. Currently supported options include: xmlns, import.
  • The third column contains IRIs. If action is replace, please provide both new IRI and old IRI separated by ';'.
  • The default header cleanup file is HERE.
